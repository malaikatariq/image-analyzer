Step-by-Step: Creating Azure Computer Vision Resource

  1. Go to the Azure Portal and sign in
  2. Click on "Create a resource" (the + icon in the top left)
  3. Search for "Computer Vision" and select it from the results
  4. Click the "Create" button
  5. Fill in the required details:
    • Name: Choose a unique name for your resource
    • Subscription: Select your Azure subscription
    • Location: Choose a region close to your users
    • Pricing tier: Select F0 (free) for testing or S1 for production
    • Resource group: Create new or select existing
  6. Click "Review + create" and then "Create"
  7. Wait for deployment to complete, then click "Go to resource"
  8. From the overview page, copy your "Endpoint" and one of your "Keys"

Azure Function Setup

Create an Azure Function to securely handle your API key:

  1. In the Azure Portal, click "Create a resource"
  2. Search for "Function App" and select it
  3. Click "Create" and fill in the details:
    • Name: Choose a unique name (e.g., "your-ocr-function")
    • Runtime stack: Node.js
    • Region: Same as your Computer Vision resource
  4. Go to your Function App after creation
  5. Click on "Configuration" in the left sidebar
  6. Add two new application settings:
    • VISION_ENDPOINT: Your Computer Vision endpoint
    • VISION_KEY: Your Computer Vision API key
  7. Create a new function:
    • Click "Functions" in the left sidebar
    • Click "Create" and select "HTTP trigger"
    • Name it "analyzeimage" and set Authorization level to "Anonymous"
  8. Replace the default code with the following:
// index.js - Azure Function code const axios = require('axios'); module.exports = async function (context, req) { context.log('JavaScript HTTP trigger function processed a request.'); // Set CORS headers for all responses const headers = { 'Content-Type': 'application/json', 'Access-Control-Allow-Origin': '*', 'Access-Control-Allow-Methods': 'GET, POST, OPTIONS', 'Access-Control-Allow-Headers': 'Content-Type' }; try { // Handle preflight OPTIONS request if (req.method === 'OPTIONS') { context.res = { status: 200, headers: headers, body: '' }; return; } context.log('Request method:', req.method); // Check if the request contains data if (!req.body) { context.log('Error: No request body found'); context.res = { status: 400, headers: headers, body: JSON.stringify({ error: "Please provide a request body with imageData" }) }; return; } context.log('Request body received:', JSON.stringify(req.body).substring(0, 200) + '...'); // Check if imageData exists in the request body if (!req.body.imageData) { context.log('Error: imageData missing from request body'); context.res = { status: 400, headers: headers, body: JSON.stringify({ error: "Please provide imageData in the request body" }) }; return; } // Get Azure Computer Vision credentials from environment variables const endpoint = process.env.VISION_ENDPOINT; const key = process.env.VISION_KEY; context.log('VISION_ENDPOINT:', endpoint ? 'Exists' : 'Missing'); context.log('VISION_KEY:', key ? 'Exists' : 'Missing'); if (!endpoint || !key) { context.log('Error: Missing environment variables'); context.res = { status: 500, headers: headers, body: JSON.stringify({ error: "Server configuration error: Vision credentials not set" }) }; return; } // Remove data URL prefix if present const imageData = req.body.imageData.replace(/^data:image\/\w+;base64,/, ''); context.log('Image data length:', imageData.length); const imageBuffer = Buffer.from(imageData, 'base64'); context.log('Image buffer length:', imageBuffer.length); // Call Azure Computer Vision OCR API context.log('Calling Azure Computer Vision API...'); const visionResponse = await axios.post( `${endpoint}/vision/v3.2/ocr`, imageBuffer, { headers: { 'Ocp-Apim-Subscription-Key': key, 'Content-Type': 'application/octet-stream' }, timeout: 30000 // 30 second timeout } ); context.log('Azure Computer Vision API call successful'); context.res = { status: 200, headers: headers, body: visionResponse.data }; } catch (error) { context.log.error('Error details:', error); // More detailed error information if (error.response) { context.log.error('Azure API response error:', error.response.status, error.response.statusText); context.log.error('Azure API response data:', error.response.data); } else if (error.request) { context.log.error('No response received from Azure API:', error.request); } else { context.log.error('Error setting up request:', error.message); } context.res = { status: 500, headers: headers, body: JSON.stringify({ error: "Failed to process image", details: error.message }) }; } };
Note: You'll also need to add CORS settings in your Function App:
  1. Go to your Function App in Azure Portal
  2. Click on "CORS" in the left sidebar
  3. Add your website URL or "*" for testing
  4. Click "Save"

GitHub Deployment for Static Web App

  1. Create a new GitHub repository
  2. Upload your HTML file (name it index.html) and create a .gitignore file with:
    # Ignore node modules node_modules/ # Ignore environment files .env
  3. In Azure Portal, create a "Static Web App" resource
  4. Connect it to your GitHub repository
  5. Configure build settings:
    • App location: / (if your HTML is in root)
    • Api location: (leave empty)
    • Output location: (leave empty)
  6. After deployment, get your Static Web App URL
  7. Update your Azure Function CORS settings to include your Static Web App URL
Important: After deploying your Azure Function, update the functionUrl variable in the JavaScript code with your actual function URL.
